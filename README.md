# Личный бред — Руслана Васильева

Лендинг-блог: обзоры красивых мест Евразии, туристические истории и фото.  
React + Vite, без бэкенда. Контент хранится в JSON.

## Запуск

```bash
npm install
npm run dev
```

Сборка: `npm run build`. Превью сборки: `npm run preview`.

## Как обновлять контент

Все данные сайта лежат в **`public/data/content.json`**. Редактируйте этот файл в любом редакторе (VS Code, Notepad и т.д.).

### Структура JSON

- **`meta`** — общие данные:
  - `siteName` — название сайта (ru, en, de).
  - `aboutMe` — текст блока «Обо мне» (ru, en, de).
  - `contacts.email` — e-mail для футера.
  - `contacts.social` — массив `{ "id": "telegram", "url": "https://...", "label": "Telegram" }` (id можно: telegram, instagram, twitter, facebook).

- **`posts`** — массив постов (первоисточник контента). Каждый пост:
  - `id` — уникальный латинский slug (например `alps-autumn`), используется в URL `/posts/alps-autumn`.
  - `type`: `"article"` (только текст) или `"articleWithPhotos"` (текст + фото).
  - `title`, `excerpt`, `body` — объекты с ключами `ru`, `en`, `de`.
  - `images` — массив URL картинок (можно внешние: CDN, облако). Для типа `article` можно `[]`. Фото из постов автоматически попадают в **галерею** — отдельный массив `gallery` в JSON не нужен.
  - `tags` — массив строк, например `["природа", "горы"]`.
  - `location` — место с переводами для ru/en/de: `{ "city": { "ru": "Кавказ", "en": "Caucasus", "de": "Kaukasus" }, "country": { "ru": "Россия", "en": "Russia", "de": "Russland" } }`. Названия мест отображаются на выбранном языке.
  - `date` — строка `YYYY-MM-DD`.
  - `featured` — `true` только у одного поста; он показывается в блоке «Главный пост» на главной.

### Как добавить новый пост

1. Откройте `public/data/content.json`.
2. В массив `posts` добавьте объект по образцу существующих (скопируйте один и измените поля).
3. Заполните `id`, `title`, `excerpt`, `body` для ru/en/de, `tags`, `location`, `date`, `type`, `images` (если есть фото), `featured` (обычно `false`).
4. Сохраните файл. После обновления страницы пост появится на сайте.

### Картинки и галерея

- **Галерея** строится автоматически из фото постов: все картинки из полей `images` постов отображаются на странице «Галерея» и в блоке галереи на главной. Отдельный массив `gallery` в JSON не используется.
- **Внешние ссылки**: вставьте полный URL в `images[]` поста (например с Unsplash, CDN или облака).
- **Локальные**: положите файлы в `public/photos/` и укажите путь вида `/photos/имя-файла.jpg` в `images[]`.

### Ограничения текста на главной

- В ленте и сетке постов показывается краткий текст: до ~30–40 слов (берётся из `excerpt` или начала `body`).
- В блоке «Главный пост» — до ~200–300 слов. Имеет смысл держать `excerpt` или начало `body` в этих пределах для главного поста.

## Языки и тема

- Язык: RU / EN / DE переключаются в шапке. Тексты из JSON подставляются по выбранному языку.
- Тема: светлая/тёмная переключается кнопкой в шапке, выбор сохраняется в браузере.

## Структура проекта

- `src/pages/` — страницы: главная, список постов, страница поста, галерея.
- `src/components/` — шапка, футер, карточка поста, блок галереи.
- `src/i18n/` — переводы интерфейса (кнопки, подписи).
- `src/data/types.ts` — типы для контента.
- `public/data/content.json` — единственный файл с контентом для правок.
